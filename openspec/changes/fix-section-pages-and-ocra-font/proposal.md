## Why

The MkDocs section index pages look bare compared to `pekkaklarck/manual` because the pipeline's CLEAN stage wipes and regenerates them with only a heading. Additionally, the OCRA font used for `h1` headings renders correctly locally (font installed on the dev machine) but falls back to a system font on CI and GitHub Pages because no font file is bundled.

## What Changes

- `reorganize.py` generates section `index.md` files with an intro paragraph and a TOC list of child pages (not just a bare `# Section Name` heading)
- `pipeline.py` CLEAN stage preserves section `index.md` files alongside the root `docs/index.md`
- OCRA font file (`OCRAStd.otf` or equivalent) is copied to `docs/assets/fonts/`
- `assets/extra.css` gains a `@font-face` declaration pointing to the bundled font file
- `mkdocs.yml` adds `docs/assets/fonts/` to watched paths (or is left as-is if `watch: docs` already covers it)

## Capabilities
