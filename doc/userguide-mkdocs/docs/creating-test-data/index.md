# Creating Test Data

This is where most of your time goes. Robot Framework data files have a clean, tabular syntax described in [Test data syntax](test-data-syntax.md) — learn it once and you're set for everything else. From there, [creating test cases](creating-test-cases.md) and [creating user keywords](creating-user-keywords.md) cover the two building blocks you'll use every day; the rest of the section fills in variables, control structures, and the finer details you'll reach for as your test suite grows.

- [Test data syntax](test-data-syntax.md) — File formats, section structure, and general parsing rules for Robot Framework data files.
- [Creating test cases](creating-test-cases.md) — How to write test cases using keywords and settings.
- [Creating test suites](creating-test-suites.md) — Organizing test cases into suites with files and directory structures.
- [Creating user keywords](creating-user-keywords.md) — Defining reusable custom keywords from existing keywords and library calls.
- [Variables](variables.md) — Built-in and user-defined variables and how to create and use them.
- [Variable files](variable-files.md) — Supplying variables from Python files for use across test suites.
- [Resource files](resource-files.md) — Sharing keywords and variables across suites through resource files.
- [Control structures](control-structures.md) — IF/ELSE, FOR, WHILE, and TRY/EXCEPT constructs for flow control.
- [Advanced features](advanced-features.md) — Template tests, metadata, name patterns, and other advanced techniques.
- [Creating tasks](creating-tasks.md) — Writing automation tasks for RPA (robotic process automation) use cases.
- [Using test libraries](using-test-libraries.md) — Importing and using standard and external keyword libraries.
